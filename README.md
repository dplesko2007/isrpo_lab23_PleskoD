## Лабораторная работа №23: Крестики-нолики (Tic-Tac-Toe)

**Автор:** Плеско Д.Д.
**Группа:** ИСП-231

#### Описание игры

Классическая игра крестики-нолики для двух игроков на поле 3×3. Игроки по очереди ставят свои символы (X и O). Побеждает тот, кто первым выстроит три своих символа в ряд — по горизонтали, вертикали или диагонали. Если все клетки заполнены и никто не победил — ничья.

#### Правила игры

•	Игрок X всегда ходит первым
•	Нельзя ходить в уже занятую клетку
•	Побеждает тот, кто закрыл линию из 3 символов
•	Кнопка "Начать заново" сбрасывает игру в начальное состояние
Особенности реализации
•	Чистый JavaScript без библиотек
•	Логика игры отделена от отображения
•	Все состояния хранятся в переменных, а не в DOM

#### Игровая механика

**Как определяется победитель?**

После каждого хода функция checkResult() перебирает все 8 выигрышных комбинаций и проверяет, заполнены ли три клетки одним символом. Если да — победитель найден.

**Как обрабатываются клики?**

Каждая клетка имеет обработчик события click, который вызывает handleCellClick(). Функция проверяет, свободна ли клетка и активна ли игра, затем записывает ход.

**Как работает перезапуск?**

Функция restartGame() сбрасывает все переменные в начальное состояние и очищает текст во всех клетках DOM.

#### Структура данных

- Массив board
Массив из 9 строк, где каждый элемент соответствует клетке поля (индексы 0–8). Пустая строка "" означает свободную клетку, "X" или "O" — занятую.

**Условия победы winConditions**

Массив из 8 вложенных массивов с индексами клеток, образующих выигрышную линию:
-	3 горизонтали: [0,1,2], [3,4,5], [6,7,8]
-	3 вертикали: [0,3,6], [1,4,7], [2,5,8]
-	2 диагонали: [0,4,8], [2,4,6]
Как хранится состояние игры?
-	currentPlayer — строка "X" или "O", чей сейчас ход
-	board — массив с содержимым каждой клетки
-	gameActive — булево значение: true = игра идёт, false = окончена

#### Алгоритмы

**Алгоритм проверки победы**

1.	Перебрать все 8 комбинаций из winConditions
2.	Для каждой взять три индекса [a, b, c]
3.	Проверить: board[a] !== "" и все три значения одинаковы
4.	Если условие выполнено — победитель найден

**Алгоритм определения ничьей**

Если после хода в массиве board не осталось пустых строк (!board.includes("")) и никто не победил — ничья.

**Алгоритм смены игрока**

Тернарный оператор переключает игрока после каждого хода:
currentPlayer = currentPlayer === "X" ? "O" : "X";
